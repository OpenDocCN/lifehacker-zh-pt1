# 何时使用条形图而不是饼图

> 原文：<https://lifehacker.com/when-to-use-bar-charts-instead-of-pie-charts-1849488600>

除非你是一名数据科学家或者有另外一份通常使用数据可视化的工作，选择正确的方式来表示信息会让感到不知所措。像 Excel 这样的软件可以将任何类型的数据填充到几乎任何类型的图表中——但是对于你想要讲述的故事，哪一个才是正确的呢？

条形图和饼图是表示数据的两种常见方式，虽然它们有时可以互换使用，但它们并不适用于每种类型的数据集。以下是如何在条形图和饼图之间做出选择。

## 何时使用条形图

条形图有两个轴:一个是类别(一年中的月份)，另一个是数值(国家公园的游客数量)。因此，你可以将一月份的访客总数与其他月份的访客总数进行比较，并观察哪些月份的访客流量最大。条形图可以水平或垂直放置。

条形图还可以用于合并数据集或显示子类别之间的比较情况，例如，按选民年龄组细分的每位候选人获得的选票数量或百分比。

## 何时使用饼图

另一方面，饼状图显示类别*占整体*的比例。您可以将圆形视为总和(100%)，每个切片的宽度由它相对于其他切片所代表的总和决定。使用类似的例子，饼图可以显示每个候选人获得的选票占所有选民投票总数的百分比。

只有当你包含的单个数据点累加成一个整体，而这个整体本身就是故事的一部分时，饼图才真正有用。如果你试图表达一个部分与整体之间的关系，例如，你的预算中有多大比例花在了租金上，那么条形图可能就不那么清楚了。

## 何时选择条形图而不是饼图

在一些情况下，尽管数据代表有意义的整体的一部分，但饼图将是一个糟糕的选择:

*   **分类太多**。如果饼图切片太窄，就很难确定它们的相对大小。此外，图表变得拥挤，标签和注释难以阅读。
*   **切片尺寸过于相似**。如果饼图块的比例接近但不相等，则相对大小可能难以解释。乍看之下，如果没有清晰的标签，饼图可能无法清楚地传达切片所代表的确切比例。
*   **您有多个数据集**。饼图只能代表一个数据集，而这个数据集又是一个整体。例如，他们无法比较按性别进一步划分的某些工作类别的工资。

虽然饼图特别擅长显示相对于整体的百分比，但条形图也可以用于显示百分比而不是绝对数字，特别是当存在上述限制或类别不相互排斥时——例如，允许用户为单个问题选择多个答案或类别的调查。

一般来说，当你比较类别时，使用条形图，当你比较部分和整体时，使用饼图。但是，如果您有许多部分(超过五个)，或者如果饼图切片的比例不容易识别(例如 1/3 或 1/4 的倍数)，条形图可能仍然更适合于部分到整体的可视化。反之亦然:如果整体没有任何意义，饼状图对于类别间的比较是没有用的。最佳底线:条形图比饼图更加通用。
